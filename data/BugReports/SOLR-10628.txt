               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Creating a collection with <tt>bin/solr create</tt> today is too verbose:</p> 
                  <div class="preformatted panel" style="border-width: 1px;">
                   <div class="preformattedContent panelContent"> 
                    <pre>$ bin/solr create -c foo

Connecting to ZooKeeper at localhost:9983 ...
INFO  - 2017-05-08 09:06:54.409; org.apache.solr.client.solrj.impl.ZkClientClusterStateProvider; Cluster at localhost:9983 ready
Uploading /Users/janhoy/git/lucene-solr/solr/server/solr/configsets/data_driven_schema_configs/conf for config foo to ZooKeeper at localhost:9983

Creating new collection 'foo' using command:
http://localhost:8983/solr/admin/collections?action=CREATE&amp;name=foo&amp;numShards=1&amp;replicationFactor=1&amp;maxShardsPerNode=1&amp;collection.configName=foo

{
  "responseHeader":{
    "status":0,
    "QTime":4178},
  "success":{"192.168.127.248:8983_solr":{
      "responseHeader":{
        "status":0,
        "QTime":2959},
      "core":"foo_shard1_replica1"}}}
</pre> 
                   </div>
                  </div> 
                  <p>A normal user don't need all this info. Propose to move all the details to verbose mode (<tt>-V)</tt> and let the default be the following instead:</p> 
                  <div class="preformatted panel" style="border-width: 1px;">
                   <div class="preformattedContent panelContent"> 
                    <pre>$ bin/solr create -c foo
Connecting to ZooKeeper at localhost:9983 ...
Created collection 'foo' with 1 shard(s), 1 replica(s) using config-set 'data_driven_schema_configs'
</pre>
