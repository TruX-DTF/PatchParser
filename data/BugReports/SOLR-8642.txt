               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Some of my colleagues and I recently noticed that the CREATECOLLECTION API will create a collection even when invalid characters are present in the name.</p> 
                  <p>For example, consider the following reproduction case, which involves creating a collection with a space in its name:</p>

                  <p>The commands/responses above suggest that Solr creates the collection without checking the name. It then goes on to create the cores for the collection, which fails and returns the error seen above.</p> 
                  <p>I verified this by doing a <tt>curl -i -l -k "http://localhost:8983/solr/admin/cores</tt>; as expected the cores were not actually created. (This is probably thanks to Erick's work on <a href="https://issues.apache.org/jira/browse/SOLR-8308" title="Core gets inaccessible after RENAME operation with special characters" class="issue-link" data-issue-key="SOLR-8308"><del>SOLR-8308</del></a>).</p> 
                  <p>This bug is a problem because it will create collections which can never be backed up with actual cores.</p> 
                  <p>Seems like the same name-verification that 8308 added to cores should also be applied to collections.</p>
