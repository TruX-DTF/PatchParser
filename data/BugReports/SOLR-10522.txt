                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>After upgrading Solr 6.3 -&gt; 6.5 I've noticed a change in how json response writer outputs "collations" response key when spellchecking is enabled (wt=json&amp;json.nl=arrarr)</p> 
                  <p>Solr 6.3:</p> 
                  <p>"collations":<br> [<br> ["collation",</p> { "collationQuery":"the", "hits":48, "maxScore":"30.282", "misspellingsAndCorrections": [ ["thea","the"]]} 
                  <p>],<br> ["collation",</p> { "collationQuery":"tea", "hits":3, "maxScore":"2.936", "misspellingsAndCorrections": [ ["thea","tea"]]} 
                  <p>],<br> ...</p> 
                  <p>Solr 6.5:</p> 
                  <p>"collations":{<br> "collation":</p> { "collationQuery":"the", "hits":43, "misspellingsAndCorrections": [ ["thea","the"]]} 
                  <p>,<br> "collation":</p> { "collationQuery":"tea", "hits":3, "misspellingsAndCorrections": [ ["thea","tea"]]} 
                  <p>,<br> ...</p> 
                  <p>Solr 6.5 outputs object instead of an array, and it has duplicate keys which is not valid for JSON format.</p> 
                  <p>Any help is appreciated.</p> 
                 </div> 
                </div> 
               </div>
