                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Solr currently has "recommendations" about what constitutes a valid identifier, but doesn't enforce these "recommendations" uniformly. Core (<a href="https://issues.apache.org/jira/browse/SOLR-8308" title="Core gets inaccessible after RENAME operation with special characters" class="issue-link" data-issue-key="SOLR-8308"><del>SOLR-8308</del></a>) and collection (<a href="https://issues.apache.org/jira/browse/SOLR-8642" title="SOLR allows creation of collections with invalid names" class="issue-link" data-issue-key="SOLR-8642"><del>SOLR-8642</del></a>) names are currently checked, but shards aren't.</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">$ bin/solr -e cloud -noprompt
    ....
$ curl -i -l -k -X GET <span class="code-quote">"http:<span class="code-comment">//localhost:8983/solr/admin/collections?action=CREATE&amp;name=coll1&amp;router.name=implicit&amp;numShards=1&amp;shards=bad+shard+name"</span>
</span>HTTP/1.1 200 OK
Content-Type: application/xml; charset=UTF-8
Transfer-Encoding: chunked

&lt;?xml version=<span class="code-quote">"1.0"</span> encoding=<span class="code-quote">"UTF-8"</span>?&gt;
&lt;response&gt;
&lt;lst name=<span class="code-quote">"responseHeader"</span>&gt;&lt;<span class="code-object">int</span> name=<span class="code-quote">"status"</span>&gt;0&lt;/<span class="code-object">int</span>&gt;&lt;<span class="code-object">int</span> name=<span class="code-quote">"QTime"</span>&gt;204&lt;/<span class="code-object">int</span>&gt;&lt;/lst&gt;&lt;lst name=<span class="code-quote">"failure"</span>&gt;&lt;str&gt;org.apache.solr.client.solrj.impl.HttpSolrClient$RemoteSolrException:Error from server at http:<span class="code-comment">//127.0.1.1:8983/solr: Error CREATEing SolrCore <span class="code-quote">'coll1_bad shard name_replica1'</span>: Unable to create core [coll1_bad shard name_replica1] Caused by: Invalid name: <span class="code-quote">'coll1_bad shard name_replica1'</span> Identifiers must consist entirely of periods, underscores and alphanumerics&lt;/str&gt;&lt;/lst&gt;
</span>&lt;/response&gt;
</pre> 
                   </div>
                  </div> 
                  <p>(Note that the CREATE command above returned 200-OK, and the failure was only apparent when viewing the message.)</p> 
                  <p>A CLUSTERSTATUS shows that the shard was actually created, but has no underlying cores.</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">$ curl -i -l -k -X GET <span class="code-quote">"http:<span class="code-comment">//localhost:8983/solr/admin/collections?action=CLUSTERSTATUS&amp;wt=json&amp;indent=<span class="code-keyword">true</span>"</span>
</span>...
    <span class="code-quote">"collections"</span>:{
      <span class="code-quote">"coll1"</span>:{
        <span class="code-quote">"replicationFactor"</span>:<span class="code-quote">"1"</span>,
        <span class="code-quote">"shards"</span>:{<span class="code-quote">"bad shard name"</span>:{
            <span class="code-quote">"range"</span>:<span class="code-keyword">null</span>,
            <span class="code-quote">"state"</span>:<span class="code-quote">"active"</span>,
            <span class="code-quote">"replicas"</span>:{}}},
        <span class="code-quote">"router"</span>:{<span class="code-quote">"name"</span>:<span class="code-quote">"implicit"</span>},
        <span class="code-quote">"maxShardsPerNode"</span>:<span class="code-quote">"1"</span>,
        <span class="code-quote">"autoAddReplicas"</span>:<span class="code-quote">"<span class="code-keyword">false</span>"</span>,
        <span class="code-quote">"znodeVersion"</span>:1,
        <span class="code-quote">"configName"</span>:<span class="code-quote">"gettingstarted"</span>},
...
</pre> 
                   </div>
                  </div> 
                  <p>This JIRA proposes adding a check to ensure that shard names meet SOLR's identifier "recommendations". This should prevent users from accidentally putting themselves in a bad state.</p>
