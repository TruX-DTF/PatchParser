                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Currently since <a href="https://issues.apache.org/jira/browse/LUCENE-2236" title="Similarity can only be set per index, but I may want to adjust scoring behaviour at a field level" class="issue-link" data-issue-key="LUCENE-2236"><del>LUCENE-2236</del></a>, we can enable Similarity per-field, but in schema.xml there is only a 'global' factory<br> for the SimilarityProvider.</p> 
                  <p>In my opinion this is too low-level because to customize Similarity on a per-field basis, you have to set your own<br> CustomSimilarityProvider with &lt;similarity class=.../&gt; and manage the per-field mapping yourself in java code.</p> 
                  <p>Instead I think it would be better if you just specify the Similarity in the FieldType, like after &lt;analyzer&gt;.</p> 
                  <p>As far as the example, one idea from <a href="https://issues.apache.org/jira/browse/LUCENE-1360" title="A Similarity class which has unique length norms for numTerms <= 10" class="issue-link" data-issue-key="LUCENE-1360">LUCENE-1360</a> was to make a "short_text" or "metadata_text" used by the<br> various metadata fields in the example that has better norm quantization for its shortness...</p> 
                 </div> 
                </div>
