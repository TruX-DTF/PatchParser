                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>When reviewing the new CharStream code added to Tokenizers, I found a<br> serious problem with backwards compatibility and other Tokenizers, that do<br> not override reset(CharStream).</p> 
                  <p>The problem is, that e.g. CharTokenizer only overrides reset(Reader):</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">  <span class="code-keyword">public</span> void reset(Reader input) <span class="code-keyword">throws</span> IOException {
    <span class="code-keyword">super</span>.reset(input);
    bufferIndex = 0;
    offset = 0;
    dataLen = 0;
  }
</pre> 
                   </div>
                  </div> 
                  <p>If you reset such a Tokenizer with another CharStream (not a Reader), this<br> method will never be called and breaking the whole Tokenizer.</p> 
                  <p>As CharStream extends Reader, I propose to remove this reset(CharStream<br> method) and simply do an instanceof check to detect if the supplied Reader<br> is no CharStream and wrap it. We could also remove the extra ctor (because<br> most Tokenizers have no support for passing CharStreams). If the ctor also<br> checks with instanceof and warps as needed the code is backwards compatible<br> and we do not need to add additional ctors in subclasses.</p> 
                  <p>As this instanceof check is always done in CharReader.get() why not remove<br> ctor(CharStream) and reset(CharStream) completely?</p> 
                  <p>Any thoughts?</p> 
                  <p>I would like to fix this somehow before RC4, I'm, sorry <img class="emoticon" src="/jira/images/icons/emoticons/sad.png" height="16" width="16" align="absmiddle" alt="" border="0"></p> 
                 </div>
