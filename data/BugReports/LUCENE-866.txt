                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>To accelerate posting list skips (TermDocs.skipTo(int)) Lucene uses skip lists. <br> The default skip interval is set to 16. If we want to skip e. g. 100 documents, <br> then it is not necessary to read 100 entries from the posting list, but only <br> 100/16 = 6 skip list entries plus 100%16 = 4 entries from the posting list. This <br> speeds up conjunction (AND) and phrase queries significantly.</p> 
                  <p>However, the skip interval is always a compromise. If you have a very big index <br> with huge posting lists and you want to skip over lets say 100k documents, then <br> it is still necessary to read 100k/16 = 6250 entries from the skip list. For big <br> indexes the skip interval could be set to a higher value, but then after a big <br> skip a long scan to the target doc might be necessary.</p> 
                  <p>A solution for this compromise is to have multi-level skip lists that guarantee a <br> logarithmic amount of skips to any target in the posting list. This patch <br> implements such an approach in the following way:</p> 
                  <p> Example for skipInterval = 3:<br> c (skip level 2)<br> c c c (skip level 1) <br> x x x x x x x x x x (skip level 0)<br> d d d d d d d d d d d d d d d d d d d d d d d d d d d d d d d d (posting list)<br> 3 6 9 12 15 18 21 24 27 30 (df)</p> 
                  <p> d - document<br> x - skip data<br> c - skip data with child pointer</p> 
                  <p>Skip level i contains every skipInterval-th entry from skip level i-1. Therefore the <br> number of entries on level i is: floor(df / ((skipInterval ^ (i + 1))).</p> 
                  <p>Each skip entry on a level i&gt;0 contains a pointer to the corresponding skip entry in <br> list i-1. This guarantees a logarithmic amount of skips to find the target document.</p> 
                  <p>Implementations details:</p> 
                  <ul> 
                   <li>I factored the skipping code out of SegmentMerger and SegmentTermDocs to<br> simplify those classes. The two new classes AbstractSkipListReader and <br> AbstractSkipListWriter implement the skipping functionality.</li> 
                   <li>While AbstractSkipListReader and Writer take care of writing and reading the<br> multiple skip levels, they do not implement an actual skip data format. The two <br> new subclasses DefaultSkipListReader and Writer implement the skip data format <br> that is currently used in Lucene (with two file pointers for the freq and prox <br> file and with payload length information). I added this extra layer to be <br> prepared for flexible indexing and different posting list formats. </li> 
                  </ul> 
                  <p>File format changes: </p> 
                  <ul> 
                   <li>I added the new parameter 'maxSkipLevels' to the term dictionary and increased the<br> version of this file. If maxSkipLevels is set to one, then the format of the freq <br> file does not change at all, because we only have one skip level as before. For <br> backwards compatibility maxSkipLevels is set to one automatically if an index <br> without the new parameter is read. </li> 
                   <li>In case maxSkipLevels &gt; 1, then the frq file changes as follows:<br> FreqFile (.frq) --&gt; &lt;TermFreqs, SkipData&gt;^TermCount<br> SkipData --&gt; &lt;&lt;SkipLevelLength, SkipLevel&gt;^(Min(maxSkipLevels, <br> floor(log(DocFreq/log(skipInterval))) - 1)&gt;, SkipLevel&gt;<br> SkipLevel --&gt; &lt;SkipDatum&gt;<sup>DocFreq/(SkipInterval</sup>(Level + 1))</li> 
                  </ul> 
                  <p> Remark: The length of the SkipLevel is not stored for level 0, because 1) it is not <br> needed, and 2) the format of this file does not change for maxSkipLevels=1 then.</p> 
                  <p>All unit tests pass with this patch.</p> 
                 </div> 
                </div> 
               </div>
