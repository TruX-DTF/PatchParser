                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Hi, </p> 
                  <p>For our workflow-system, we can use a variety of DBMS as the store engine. Derby is one of them especially popular<br> for test and development purposes. During setup of our system, we allow to upload a jar file with the JDBC-Driver for the DBMS.<br> In case of derby, we use the embedded mode and upload derby.jar. After that we add the jar to the repositories of our class loader<br> and call </p> 
                  <p>Class drc = Class.forName("org.apache.derby.jdbc.EmbeddedDriver", true, ourclassloader);<br> Driver drv = (Driver)drc.newInstance();</p> 
                  <p>this worked perfectly for many years up to version 10.6.1.0, but ceased to work in 10.6.2.1; <br> i assume the unreleased future versions are affected, too</p> 
                  <p>when derby.jar is placed in the classpath manually before starting our setup, everything works fine regardless of the derby version.</p> 
                  <p>investigations lead to the following conclusion:</p> 
                  <ul class="alternate" type="square"> 
                   <li>since rev. 982370 this is broken, in the immediate predecessor rev. 980035 it worked.</li> 
                   <li>rev. 982370 in the org.apache.derby.impl.store.raw.data.BaseDataFileFactory class introduced a new method<br> private static String jarClassPath(final Class cls) which tries to find the jarfile from which derby was loaded.</li> 
                  </ul> 
                  <p> it contains the following lines: </p> 
                  <p>if ( cs == null )<br> return null;<br> URL result = cs.getLocation();<br> return result.toString(); </p> 
                  <p> but in the case, when we dynamically load derby, <br> cs is not null but cs.getLocation() is null. so i propose to change the line with the if to:<br> ...<br> if ( cs == null || cs.getLocation()==null)<br> return null;<br> ...</p> 
                  <p>or maybe it would be better to surround the whole body of the method with a try catch, since it is not essential, and a null <br> return value is also ok?</p> 
                  <p>thank you for your efforts concerning derby at large, </p> 
                  <p>Michael</p> 
                 </div> 
                </div> 
               </div>
              </div>
