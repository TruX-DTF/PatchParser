                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>With empty ant.properties and JDK 6 installed in /tmp/jdk as the only JDK on the system, PropertySetter complains that it's not able to set java16compile.classpath:</p> 
                  <p>,----</p> 
                  <div class="table-wrap"> 
                   <table class="confluenceTable">
                    <tbody> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span></td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> PropertySetter environment =</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span></td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> java.vendor = Sun Microsystems Inc.</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> java.home = /tmp/jdk/jre</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> java.version = 1.6.0_13</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> os.name = SunOS</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> j14lib = null</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> j15lib = null</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span> j16lib = null</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> <span class="error">[setJdkProperties]</span></td> 
                     </tr> 
                     <tr> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> BUILD FAILED</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> /code/derby/trunk0/build.xml:203: Don't know how to set java15compile.classpath, java16compile.classpath using this environment:</td> 
                     </tr> 
                     <tr> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> java.vendor = Sun Microsystems Inc.</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> java.home = /tmp/jdk/jre</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> java.version = 1.6.0_13</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> os.name = SunOS</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> j14lib = null</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> j15lib = null</td> 
                     </tr> 
                     <tr> 
                      <td class="confluenceTd"> j16lib = null<br> `----</td> 
                     </tr> 
                    </tbody>
                   </table> 
                  </div> 
                  <p>Since it is able to detect both that the version of the default JDK is 1.6.0_13 and where it is installed, setting java16compile.classpath should be trivial.</p> 
                  <p>If the name of the JDK directory is changed from /tmp/jdk to /tmp/jdk1.6.0, Derby is built successfully.</p>
