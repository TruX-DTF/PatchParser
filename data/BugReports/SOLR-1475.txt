               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The issue title reflects Mark Miller's initial diagnosis of the problem.</p> 
                  <p>Here are my symptoms:</p> 
                  <p>This is regarding the backup feature of replication, as opposed to replication. Backups seem to work fine on toy indexes. When trying backups out on a copy of my production index (300GB-ish), though, I'm getting FileNotFoundExceptions. These cancel the backup, and delete the snapshot.yyyymmdd* directory. It seems reproducible, in that every time I try to make a backup of my large index it will fail the same way.</p> 
                  <p>This is Solr r815830. I'm not sure if this is something that would potentially be addressed by <a href="https://issues.apache.org/jira/browse/SOLR-1458" title="Java Replication error: NullPointerException SEVERE: SnapPull failed on 2009-09-22 nightly" class="issue-link" data-issue-key="SOLR-1458"><del>SOLR-1458</del></a>? (That patch is from after r815830.)</p> 
                  <p>For now I'm not using any event-based backup triggers; instead I'm manually hitting</p> 
                  <p><a href="http://master_host:port/solr/replication?command=backup" class="external-link" rel="nofollow">http://master_host:port/solr/replication?command=backup</a></p> 
                  <p>This successfully sets off a snapshot, as seen in a thread dump. However, after a while the snapshot fails. I'll paste in a couple of stack traces below.</p> 
                  <p>I haven't seen any other evidence that my index is corrupt; in particular, searching the index and Java-based replication seem to be working fine, and the Lucene CheckIndex tool did not report any problems with the index.</p>
