               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Below is a new version of FSDirectory.java. It is based on 1/30/2003 source.</p> 
                  <p>I have added one new version each of getDirectory(File) and getDirectory<br> (String). They take a new third argument 'boolean useLocks'.</p> 
                  <p>The previous 'private static final boolean DISABLE_LOCKS' has been changed <br> to 'private static boolean _disableLocks' and is initialized to false. I also <br> added a new method 'private boolean locksDisabledByProp' that checks <br> the 'disableLuceneLocks' system property.</p> 
                  <p>Method makeLock now checks the static _disableLocks as the first term in an OR <br> clause, the second of which is a call to locksDisabledByProp. This allows use <br> in an applet that does not have write access to the local filesystem, and when <br> the API is used in that way prevents the query of the system property that is <br> also disallowed in an applet by default (at least in Mozilla/Netscape).</p> 
                  <p>From my applet, I can now invoke:</p> 
                  <p>Searcher searcher = new IndexSearcher(<br> IndexReader.open(FSDirectory.getDirectory(indexFile, false, false)));</p>
