                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The current update processor chain is rather cumbersome and we should be able to use the updateprocessors without a chain.</p> 
                  <p>The scope of this ticket is </p> 
                  <ul> 
                   <li>A new tag <tt>&lt;updateProcessor&gt;</tt> becomes a toplevel tag and it will be equivalent to the <tt>&lt;processor&gt;</tt> tag inside <tt>&lt;updateRequestProcessorChain&gt;</tt> . The only difference is that it should require a <tt>name</tt> attribute. The <tt>&lt;updateProcessorChain&gt;</tt> tag will continue to exist and it should be possible to define <tt>&lt;processor&gt;</tt> inside as well . It should also be possible to reference a named URP in a chain.</li> 
                   <li>processors will be added in the request with their names . Example <tt>processor=a,b,c</tt> , <tt>post-processor=x,y,z</tt> . This creates an implicit chain of the named URPs the order they are specified</li> 
                   <li>There are multiple request parameters supported by update request 
                    <ul> 
                     <li>processor : This chain is executed executed at the leader right before the LogUpdateProcessorFactory + DistributedUpdateProcessorFactory . The replicas will not execute this.</li> 
                     <li>post-processor : This chain is executed right before the RunUpdateProcessor in all replicas , including the leader</li> 
                    </ul> </li> 
                   <li>What happens to the update.chain parameter ? <tt>update.chain</tt> will be honored . The implicit chain is created by merging both the update.chain and the request params. <tt>post-processor</tt> will be inserted right before the <tt>RunUpdateProcessorFactory</tt> in the chain. and <tt>processor</tt> will be inserted right before the LogUpdateProcessorFactory,DistributedUpdateProcessorFactory</li> 
                  </ul> 
                 </div> 
                </div>
