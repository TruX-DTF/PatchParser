                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>In discussion on derby-dev regarding how much memory is used for hash joins, Knut noted:</p> 
                  <p>I haven't verified, but I think HashJoinStrategy uses<br> DataTypeDescriptor.estimatedMemoryUsage() to estimate how much memory<br> the hash table will consume. That method has no case for BLOB or CLOB,<br> so it looks as if it will return zero for LOB columns. If that's so, it<br> will definitely overestimate how many rows fits in maxMemoryPerTable<br> kilobytes if the rows contain LOBs.</p> 
                  <p>DataTypeDescriptor.estimatedMemoryUsage() should be updated to include BLOB and CLOB and we should try verify if this theory is correct with a reproduction.</p> 
                 </div> 
                </div>
