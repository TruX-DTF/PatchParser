                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>While reviewing the warnings (<a href="https://issues.apache.org/jira/browse/LUCENE-6740" title="Reduce warnings emitted by javac in Java 7/Java 8" class="issue-link" data-issue-key="LUCENE-6740">LUCENE-6740</a>) I noticed the following:<br> The LovinsStemmer and the FinnishStemmer from the snowball package cannot work at all. Indeed Robert Muir added a comment to the test referring to: <a href="http://article.gmane.org/gmane.comp.search.snowball/1139" class="external-link" rel="nofollow">http://article.gmane.org/gmane.comp.search.snowball/1139</a></p> 
                  <p>The bug is the following: The Among class looks up the method to call via reflection. But the stemmer compiler creates all these methods as private. As AccessibleObject#setAccessible() is called nowhere (we also don't want this), the SnowballProgram main class cannot call the method. Unfortunately it completely supresses all reflection errors and returns false. SnowballProgram then thinks, the called <tt>private boolean r_A()</tt> method returned false and so the whole snowball algorithm breaks.</p> 
                  <p>Also the snowball stemmer classes had a bug: methodObject is a static instance of the Stemmer, and Among calls the private method then on wrong object. So when fixing the above, this would fail again (because the stemmer changes a static singleton object, not itsself!). We also need to change the object in SnowballProgram to use "this" when calling the method.</p> 
                  <p>There are several possibilities to solve the private methods problem:</p> 
                  <ul class="alternate" type="square"> 
                   <li>call <tt>AccessibleObject.setAccessible(true)</tt> -&gt; we don't want this! Never Ever! <img class="emoticon" src="/jira/images/icons/emoticons/smile.png" height="16" width="16" align="absmiddle" alt="" border="0"></li> 
                   <li>patch the whole Snowball-generated classes to make those methods "public". This is a huge patch and shows many internals =&gt; no way</li> 
                   <li>Since Java 7 we can use a cool trick, my favourite: MethodHandles</li> 
                  </ul> 
                  <p>The MethodHandles trick can be done the following way:</p> 
                  <ul class="alternate" type="square"> 
                   <li>Change the Among class to get a <tt>MethodHandles.Lookup</tt> in its ctor (instead of the broken static <tt>SnowballProgram</tt> instance <tt>methodObject</tt>). This lookup is used to lookup the method by name. This Lookup Object must be private to our implementing class (the one that defines the private methods). If this is the case, we can get a method handle and call the method (from anywhere, because we "own" it).</li> 
                   <li>replace the (already broken) <tt>methodObject</tt> in every stemmer to instead be the correct <tt>MethodHandles.Lookup</tt> of the implementing class containing the private method. This has 2 effects: The class to get the lookup gets all access rights to the class calling it (so we can access private methods) and also sees all methods.</li> 
                  </ul> 
                  <p>The second part is done by a new Ant task: <tt>ant patch-snowball</tt>. Whenever you add a new Snowball stemmer, copy the java file generated by the snowball compiler to the ext directory and call the ant task. It will patch the <tt>methodObject</tt> declaration (and also add a <tt>@SuppressWarnings("unused")</tt> for convenience).</p> 
                  <p>I reenabled the dictionary tests and the whole stemmer works. There are also no issues with security managers, because we do nothing security sensitive (all is our own stuff, no setAccessible).</p> 
                  <p>And finally: LovinsStemmer and FinnishStemmer got insanely fast (and correct, of course).</p>
