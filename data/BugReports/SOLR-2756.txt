                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I included a SolrJ 3.3 dependency into a new project and I noticed needless dependencies transitive show up.</p> 
                  <p>Here is a subset of the output from mvn dependency:tree:</p> 
                  <div class="preformatted panel" style="border-width: 1px;">
                   <div class="preformattedContent panelContent"> 
                    <pre>
[INFO] +- org.apache.solr:solr-solrj:jar:3.3.0:compile
[INFO] |  +- org.apache.lucene:lucene-core:jar:3.3.0:compile
[INFO] |  +- commons-httpclient:commons-httpclient:jar:3.1:compile
[INFO] |  |  \- commons-codec:commons-codec:jar:1.2:compile
[INFO] |  +- org.apache.geronimo.specs:geronimo-stax-api_1.0_spec:jar:1.0.1:compile
[INFO] |  +- org.apache.zookeeper:zookeeper:jar:3.3.1:compile
[INFO] |  |  +- log4j:log4j:jar:1.2.15:compile
[INFO] |  |  |  \- javax.mail:mail:jar:1.4:compile
[INFO] |  |  |     \- javax.activation:activation:jar:1.1:compile
[INFO] |  |  \- jline:jline:jar:0.9.94:compile
[INFO] |  \- org.codehaus.woodstox:wstx-asl:jar:3.2.7:compile
[INFO] |     \- stax:stax-api:jar:1.0.1:compile
</pre> 
                   </div>
                  </div> 
                  <p>Clearly there is an inconsistency with solr/dist/solrj-lib and this list.</p> 
                  <ul> 
                   <li>lucene-core dependency should be removed</li> 
                   <li>AFAIK, geronimo-stax-api and wstx-asl are only needed for Java 1.5. Right? These can be put in a maven profile activated by jdk1.5.</li> 
                   <li>zookeeper dependency should be removed. Is this used in Solr 4? Even if it is, it strikes me as an optional dependency.</li> 
                  </ul> 
                 </div> 
                </div> 
               </div>
