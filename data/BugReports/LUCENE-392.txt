                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I am not sure if this is because Lucene historically needed to work with older<br> JVM's but with modern JVM's, instanceof is much quicker. </p> 
                  <p>The Field.stringValue(), .readerValue(), and .binaryValue() methods all use<br> ClassCastException checking.</p> 
                  <p>Using the following test-bed class, you will see that instanceof is miles quicker:</p> 
                  <p>package com.aconex.index;</p> 
                  <p>public class ClassCastExceptionTest {</p> 
                  <p> private static final long ITERATIONS = 100000;</p> 
                  <p> /**</p> 
                  <ul> 
                   <li>@param args<br> */<br> public static void main(String[] args) { runClassCastTest(1); // once for warm up runClassCastTest(2); runInstanceOfCheck(1); runInstanceOfCheck(2); } <p> private static void runInstanceOfCheck(int run) {<br> long start = System.currentTimeMillis();</p></li> 
                  </ul> 
                  <p> Object foo = new Foo();<br> for (int i = 0; i &lt; ITERATIONS; i++) {<br> String test;<br> if(foo instanceof String) </p> { System.out.println("Is a string"); // should never print } 
                  <p> }<br> long end = System.currentTimeMillis();<br> long diff = end - start;<br> System.out.println("InstanceOf checking run #" + run + ": " + diff + "ms");</p> 
                  <p> }</p> 
                  <p> private static void runClassCastTest(int run) {<br> long start = System.currentTimeMillis();</p> 
                  <p> Object foo = new Foo();<br> for (int i = 0; i &lt; ITERATIONS; i++) {<br> String test;<br> try </p> { test = (String)foo; } 
                  <p> catch (ClassCastException c) </p> { // ignore } 
                  <p> }<br> long end = System.currentTimeMillis();<br> long diff = end - start;<br> System.out.println("ClassCast checking run #" + run + ": " + diff + "ms");<br> }</p> 
                  <p> private static final class Foo {<br> }</p> 
                  <p>}</p> 
                  <p>Results<br> =======</p> 
                  <p>Run #1</p> 
                  <p>ClassCast checking run #1: 1660ms<br> ClassCast checking run #2: 1374ms<br> InstanceOf checking run #1: 8ms<br> InstanceOf checking run #2: 4ms</p> 
                  <p>Run #2<br> ClassCast checking run #1: 1280ms<br> ClassCast checking run #2: 1344ms<br> InstanceOf checking run #1: 7ms<br> InstanceOf checking run #2: 2ms</p> 
                  <p>Run #3<br> ClassCast checking run #1: 1347ms<br> ClassCast checking run #2: 1250ms<br> InstanceOf checking run #1: 7ms<br> InstanceOf checking run #2: 2ms</p> 
                  <p>This could explain why Documents with more Fields scales worse, as in, for lots<br> of Documents with lots of Fields, the effect is exacerbated.</p> 
                 </div> 
                </div>
