                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I've worked out the start at a new postings format that should have<br> less overhead for fixed-int[] encoders (For,PFor)... using ideas from<br> the old bulk branch, and new ideas from Robert.</p> 
                  <p>It's only a start: there's no payloads support yet, and I haven't run<br> Lucene's tests with it, except for one new test I added that tries to<br> be a thorough PostingsFormat tester (to make it easier to create new<br> postings formats). It does pass luceneutil's performance test, so<br> it's at least able to run those queries correctly...</p> 
                  <p>Like Lucene40, it uses two files (though once we add payloads it may<br> be 3). The .doc file interleaves doc delta and freq blocks, and .pos<br> has position delta blocks. Unlike sep, blocks are NOT shared across<br> terms; instead, it uses block encoding if there are enough ints to<br> encode, else the same Lucene40 vInt format. This means low-freq terms<br> (&lt; 128 = current default block size) are always vInts, and high-freq<br> terms will have some number of blocks, with a vInt final block.</p> 
                  <p>Skip points are only recorded at block starts.</p> 
                 </div>
