                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>This is a first step (nowhere near committable!), implementing the<br> design iterated to in the recent "Baby steps towards making Lucene's<br> scoring more flexible" java-dev thread.</p> 
                  <p>The idea is (if you turn it on for your Field; it's off by default) to<br> store full stats in the index, into a new _X.sts file, per doc (X<br> field) in the index.</p> 
                  <p>And then have FieldSimilarityProvider impls that compute doc's boost<br> bytes (norms) from these stats.</p> 
                  <p>The patch is able to index the stats, merge them when segments are<br> merged, and provides an iterator-only API. It also has starting point<br> for per-field Sims that use the stats iterator API to compute boost<br> bytes. But it's not at all tied into actual searching! There's still<br> tons left to do, eg, how does one configure via Field/FieldType which<br> stats one wants indexed.</p> 
                  <p>All tests pass, and I added one new TestStats unit test.</p> 
                  <p>The stats I record now are:</p> 
                  <ul class="alternate" type="square"> 
                   <li>field's boost</li> 
                  </ul> 
                  <ul class="alternate" type="square"> 
                   <li>field's unique term count (a b c a a b --&gt; 3)</li> 
                  </ul> 
                  <ul class="alternate" type="square"> 
                   <li>field's total term count (a b c a a b --&gt; 6)</li> 
                  </ul> 
                  <ul class="alternate" type="square"> 
                   <li>total term count per-term (sum of total term count for all docs<br> that have this term)</li> 
                  </ul> 
                  <p>Still need at least the total term count for each field.</p>
