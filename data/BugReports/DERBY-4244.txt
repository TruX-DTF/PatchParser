                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>While working with Eranda on <a href="https://issues.apache.org/jira/browse/DERBY-4187" title="Convert altertable.sql to JUnit" class="issue-link" data-issue-key="DERBY-4187"><del>DERBY-4187</del></a>, I stumbled across an apparent ALTER TABLE bug.</p> 
                  <p>Here's a script which reproduces the problem for me:</p> 
                  <p>autocommit off;<br> create table t0(c1 int not null constraint p1 primary key);<br> alter table t0 add column c1 int;<br> alter table t0 add column c2 int not null default 0 primary key;<br> alter table t0 add column c2 int not null default 0;</p> 
                  <p>The "autocommit off" is crucial; otherwise the problem does not reproduce.</p> 
                  <p>Here's the detailed assertion failure:</p> 
                  <p>2009-05-23 15:01:17.436 GMT Thread<span class="error">[main,5,main]</span> (XID = 146), (SESSIONID = 1), (DATABASE = brydb), (DRDAID = null), Failed Statement is: alter table t0 add column c2 int not null default 0<br> org.apache.derby.shared.common.sanity.AssertFailure: ASSERT FAILED column_id = 1format_ids.length = 2format_ids = [I@1321f5<br> at org.apache.derby.shared.common.sanity.SanityManager.THROWASSERT(SanityManager.java:162)<br> at org.apache.derby.shared.common.sanity.SanityManager.THROWASSERT(SanityManager.java:147)<br> at org.apache.derby.impl.store.access.heap.Heap.addColumn(Heap.java:418)<br> at org.apache.derby.impl.store.access.RAMTransaction.addColumnToConglomerate(RAMTransaction.java:618)<br> at org.apache.derby.impl.sql.execute.AlterTableConstantAction.addNewColumnToTable(AlterTableConstantAction.java:1325)<br> at org.apache.derby.impl.sql.execute.AlterTableConstantAction.executeConstantAction(AlterTableConstantAction.java:449)</p> 
                  <p>Here's the relevant section of Heap.java:</p> 
                  <p> if (column_id != format_ids.length)</p> { if (SanityManager.DEBUG) SanityManager.THROWASSERT( "column_id = " + column_id + "format_ids.length = " + format_ids.length + "format_ids = " + format_ids); throw(StandardException.newException( SQLState.HEAP_TEMPLATE_MISMATCH, new Long(column_id), new Long(this.format_ids.length))); } 
                 </div> 
                </div> 
               </div>
