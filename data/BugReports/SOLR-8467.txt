               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Currently, it's impossible to, say, specify multiple "fq" clauses when using Streaming Aggregation due to the fact that the c'tors take a Map of params.</p> 
                  <p>Opening to discuss whether we should<br> 1&gt; deprecate the current c'tor<br> and/or<br> 2&gt; add a c'tor that takes a SolrParams object instead.<br> and/or<br> 3&gt; ???</p> 
                  <p>I don't see a clean way to go from a Map&lt;String, String&gt; to a (Modifiable)SolrParams, so existing code would need a significant change. I hacked together a PoC, just to see if I could make CloudSolrStream take a ModifiableSolrParams object instead and it passes tests, but it's so bad that I'm not going to even post it. There's <em>got</em> to be a better way to do this, but at least it's possible....</p> 
                 </div>
