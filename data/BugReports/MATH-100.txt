                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Hi, for all that might use this class:</p> 
                  <p>several things I found when using this class to calculate the<br> cumulative probability. I attached my code FYI. three things:</p> 
                  <p>1. when I used my code to calculate the cumulativeProbability(50) of<br> 5000 200 100 (Population size, number of successes, sample size),<br> result was greater than 1 (1.0000000000134985);<br> 2. when I calculated cumulativeProbability(50) and<br> cumulativeProbability(51) for the distribution 5000 200 100, I got the<br> same results, but it should have been different;<br> 2. the cumulativeProbability<img class="emoticon" src="/jira/images/icons/emoticons/error.png" height="16" width="16" align="absmiddle" alt="" border="0"> returns "for this distribution, X,<br> P(X&lt;=x)", but most of the time (at least in my case) what I care about<br> is the upper tail (X&gt;=x). based on the above findings, I can't simply<br> use 1-cumulativeProbability(x-1) to get what I want.</p> 
                  <p>here's what I think might be related to the problem: since the<br> cumulativeProbability is calculating the lower tail (X&lt;=x), a<br> distribution like above often has this probability very close to 1;<br> thus it's difficult to record a number that = 1-1E-50 'cause all you<br> can do is record sth like 0.9999..... and further digits will be<br> rounded. to avoid this, I suggest adding a new function to calculate<br> upper tail or change this to calculate x in a range like (n&lt;=x&lt;=m), in<br> addition to fix the overflow of the current function.</p> 
                  <p>thank you for your patience to get here.
