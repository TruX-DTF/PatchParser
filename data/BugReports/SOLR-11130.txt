                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I am trying to use V2Request to invoke the config API with SolrJ. Sometimes, I see a weird NullPointerException:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">5396 ERROR (qtp1277924867-37) [n:127.0.0.1:33527_solr    ] o.a.s.s.HttpSolrCall <span class="code-keyword">null</span>:java.lang.NullPointerException
	at org.apache.solr.servlet.HttpSolrCall.call(HttpSolrCall.java:518)
	at org.apache.solr.servlet.SolrDispatchFilter.doFilter(SolrDispatchFilter.java:378)
	at org.apache.solr.servlet.SolrDispatchFilter.doFilter(SolrDispatchFilter.java:322)
</pre> 
                   </div>
                  </div> 
                  <p>and the corresponding exception on the solrj client was:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">org.apache.solr.client.solrj.impl.HttpSolrClient$RemoteExecutionException: Error from server at http:<span class="code-comment">//127.0.0.1:33527/solr: Unknown Error
</span>
	at __randomizedtesting.SeedInfo.seed([33AAA14CC3DE7EE3:BBFE9E966D22131B]:0)
	at org.apache.solr.client.solrj.impl.HttpSolrClient$RemoteExecutionException.create(HttpSolrClient.java:812)
	at org.apache.solr.client.solrj.impl.HttpSolrClient.executeMethod(HttpSolrClient.java:602)
	at org.apache.solr.client.solrj.impl.HttpSolrClient.request(HttpSolrClient.java:252)
	at org.apache.solr.client.solrj.impl.HttpSolrClient.request(HttpSolrClient.java:241)
	at org.apache.solr.client.solrj.impl.LBHttpSolrClient.doRequest(LBHttpSolrClient.java:483)
	at org.apache.solr.client.solrj.impl.LBHttpSolrClient.request(LBHttpSolrClient.java:413)
	at org.apache.solr.client.solrj.impl.CloudSolrClient.sendRequest(CloudSolrClient.java:1121)
	at org.apache.solr.client.solrj.impl.CloudSolrClient.requestWithRetryOnStaleState(CloudSolrClient.java:862)
	at org.apache.solr.client.solrj.impl.CloudSolrClient.request(CloudSolrClient.java:793)
	at org.apache.solr.client.solrj.SolrClient.request(SolrClient.java:1219)
	at org.apache.solr.cloud.TestCloudSearcherWarming.test(TestCloudSearcherWarming.java:68)
</pre> 
                   </div>
                  </div> 
                  <p>The code used to invoke the api was:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">V2Request request = <span class="code-keyword">new</span> V2Request.Builder(<span class="code-quote">"/c/"</span> + collectionName + <span class="code-quote">"/config"</span>).withMethod(SolrRequest.METHOD.POST).withPayload(addListenerCommand).build();
    solrClient.request(request);
</pre> 
                   </div>
                  </div> 
                  <p>This exception happens when the config API call is sent to a Solr node which does not have any replica for the collection. Now in v1 API, solrj would refuse to send the request and complain that no default collection is set but the V2Request does not do that.</p> 
                  <p>So there are two bugs here:</p> 
                  <ol> 
                   <li>V2Request tries to send request for collection X to a random node without caring for correct routing</li> 
                   <li>The node receiving such request is not able to forward it to the right node and fails with a NPE.</li> 
                  </ol> 
                  <p>To solve 1, our options are:</p> 
                  <ol> 
                   <li>Either we start requiring the same for V2 i.e. user must set default collection</li> 
                   <li>We detect that users are trying to invoke a collection specific v2 api and we add a new method in V2RequestBuilder to specify a collection name and ensure that the user calls it.</li> 
                  </ol> 
                 </div> 
                </div>
