                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>It would be useful to be able to register handlers after SolrCore has been initialized initialized. It is also useful to be able to ask what handlers are registered and to where. This patch adds the following functions to SolrCore:</p> 
                  <p>SolrRequestHandler registerRequestHandler(String handlerName, SolrRequestHandler handler);<br> Collection&lt;SolrRequestHandler&gt; getRequestHandlers(Class&lt;? extends SolrRequestHandler&gt; clazz);</p> 
                  <p>It also guarantees that request handlers will be initialized with an argument saying what path it is registered to. RequestHandlerBase gets a bean for the registered path.</p> 
                  <p>While discussing this, Yonik suggested making it possible to defer initialization of some handlers that will be infrequently used. I added the 'LazyRequestHandlerWrapper' (if taking this out makes the patch any easier to commit - it can get its own issue)</p> 
                  <p>check:<br> <a href="http://www.nabble.com/Dynamic-RequestHandler-loading-tf3351707.html" class="external-link" rel="nofollow">http://www.nabble.com/Dynamic-RequestHandler-loading-tf3351707.html</a></p> 
                 </div> 
                </div>
