                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Getting an error (using Derby 10.4.1.3) when deleting records from a table using a qualified synonymn name.<br> Table Name is ABC_PROCESS_INVOCATION_XML and the synonmyn name is SYN_ABC_P_I_X</p> 
                  <p>The query being fired was<br> "Delete from SYN_ABC_P_I_X where ID = ? "</p> 
                  <p>from the error log<br> ---------------<br> Caused by: ERROR 42X04: Column 'APP.ABC_PROCESS_INVOCATION_XML.ID' is either not in any table in the FROM list or appears within a join specification and is outside the scope of the join specification or appears in a HAVING clause and is not in the GROUP BY list. If this is a CREATE or ALTER TABLE statement then 'APP.ABC_PROCESS_INVOCATION_XML.ID' is not a column in the target table.<br> ---------------</p> 
                  <p>After going thru few earlier bugs, a similar bug has been logged earlier bug <a href="https://issues.apache.org/jira/browse/DERBY-1784" title="When qualifying a column name with its synonym name in the select list, Derby throws SQLSTATE 42X04." class="issue-link" data-issue-key="DERBY-1784"><del>DERBY-1784</del></a> <br> <a href="http://issues.apache.org/jira/browse/DERBY-1784" class="external-link" rel="nofollow">http://issues.apache.org/jira/browse/DERBY-1784</a></p> 
                  <p>But looks like the fix did not make its way to 10.4.1.3?</p> 
                 </div> 
                </div> 
               </div>
