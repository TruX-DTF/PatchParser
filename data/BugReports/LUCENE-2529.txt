                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I'm doing some fancy stuff with span queries that is very sensitive to term positions. I discovered that the position increment gap on indexing is only applied between values when there are existing terms indexed for the document. I suspect this logic wasn't deliberate, it's just how its always been for no particular reason. I think it should always apply the gap between fields. Reference DocInverterPerField.java line 82:</p> 
                  <p>if (fieldState.length &gt; 0)<br> fieldState.position += docState.analyzer.getPositionIncrementGap(fieldInfo.name);</p> 
                  <p>This is checking fieldState.length. I think the condition should simply be: if (i &gt; 0).<br> I don't think this change will affect anyone at all but it will certainly help me. Presently, I can either change this line in Lucene, or I can put in a hack so that the first value for the document is some dummy value which is wasteful.</p> 
                 </div>
