                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Currently SolrHttpClientBuilder needs to be configured via invoking HttpClientUtil#setHttpClientBuilder(...) API. On the other hand SolrCLI attempts to support configuring SolrHttpClientBuilder via Java system property. <a href="https://github.com/apache/lucene-solr/blob/9f58b6cd177f72b226c83adbb965cfe08d61d2fb/solr/core/src/java/org/apache/solr/util/SolrCLI.java#L265" class="external-link" rel="nofollow">https://github.com/apache/lucene-solr/blob/9f58b6cd177f72b226c83adbb965cfe08d61d2fb/solr/core/src/java/org/apache/solr/util/SolrCLI.java#L265</a></p> 
                  <p>But after changes for <a href="https://issues.apache.org/jira/browse/SOLR-4509" title="Move to non deprecated HttpClient impl classes to remove stale connection check on every request and move connection lifecycle management towards the client." class="issue-link" data-issue-key="SOLR-4509"><del>SOLR-4509</del></a>, this is no longer working. This is because we need to configure HttpClientBuilderFactory which can provide appropriate SolrHttpClientBuilder instance (e.g. Krb5HttpClientBuilder). I verified that SolrCLI does not work in a kerberos enabled cluster. During the testing I also found that SolrCLI is hardcoded to use basic authentication,</p> 
                  <p><a href="https://github.com/apache/lucene-solr/blob/9f58b6cd177f72b226c83adbb965cfe08d61d2fb/solr/core/src/java/org/apache/solr/util/SolrCLI.java#L156" class="external-link" rel="nofollow">https://github.com/apache/lucene-solr/blob/9f58b6cd177f72b226c83adbb965cfe08d61d2fb/solr/core/src/java/org/apache/solr/util/SolrCLI.java#L156</a></p> 
                  <p>This jira is to add support for configuring HttpClientBuilderFactory as a java system property so that SolrCLI as well as other Solr clients can also benefit this. Also we should provide a HttpClientBuilderFactory which support configuring preemptive basic authentication so that we can remove the hardcoded basic auth usage in SolrCLI (and enable it work with kerberos). </p> 
                 </div>
