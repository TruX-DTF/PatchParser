                  <p>To me it looks like there is something wrong with the test runner. In other cases this totally-hanging threads were caused by permgen-errors, but nothing like this in the logs. It looks like the slave is waiting forever to get some messages from the master or vice versa.</p> 
                  <p>You can imitate clover tests on ASF Jenkins by running:</p> 
                  <div class="preformatted panel" style="border-width: 1px;">
                   <div class="preformattedContent panelContent"> 
                    <pre>ANT_OPTS="-Xmx1536M" ant -Dtests.jettyConnector=Socket jenkins-clover
</pre> 
                   </div>
                  </div> 
                  <p>(the ANT_OPTS are just for the generate-clover-task using lots of memory - totally unrelated. This is just needed after the tests ran successfully). jenkins-clover calls run-clover task which automatically gives more memory to the child JVMs, so maybe this is related and needs more memory. You can find that in the main common-build.xml file, where you will find a property definition if="run.clover" giving more heap and code cache to childs:</p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-xml">  <span class="code-tag">&lt;condition property=<span class="code-quote">"tests.heapsize"</span> value=<span class="code-quote">"768M"</span>&gt;</span>

  <span class="code-tag">&lt;/condition&gt;</span>
  <span class="code-tag">&lt;property name=<span class="code-quote">"tests.clover.args"</span> value=""/&gt;</span>
</pre> 
                   </div>
                  </div> 
                  <p>(maybe tweak that if Solr needs more memory - Lucene clover passes fine).</p> 
                  <p>I disabled clover tests for now.</p>
