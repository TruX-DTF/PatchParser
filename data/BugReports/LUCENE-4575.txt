               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Spinoff from here <a href="http://lucene.472066.n3.nabble.com/commit-with-only-commitData-td4022155.html" class="external-link" rel="nofollow">http://lucene.472066.n3.nabble.com/commit-with-only-commitData-td4022155.html</a>.</p> 
                  <p>In some cases, it is valuable to be able to commit changes to the index, even if the changes are just commitData. Such data is sometimes used by applications to register in the index some global application information/state.</p> 
                  <p>The proposal is:</p> 
                  <ul> 
                   <li>Add a setCommitData() API and separate it from commit() and prepareCommit() (simplify their API)</li> 
                  </ul> 
                  <ul> 
                   <li>When that API is called, flip on the dirty/changes bit, so that this gets committed even if no other changes were made to the index.</li> 
                  </ul> 
                  <p>I will work on a patch a post.</p> 
                 </div>
