                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>With current trunk NGramTokenFilter(min=2,max=4) , I index "abcdef" string into an index, but I can't query it with "abc". If I query with "ab", I can get a hit result.</p> 
                  <p>The reason is that the NGramTokenFilter generates badly ordered TokenStream. Query is based on the Token order in the TokenStream, that how stemming or phrase should be anlayzed is based on the order (Token.positionIncrement).</p> 
                  <p>With current filter, query string "abc" is tokenized to : ab bc abc <br> meaning "query a string that has ab bc abc in this order".<br> Expected filter will generate : ab abc(positionIncrement=0) bc<br> meaning "query a string that has (ab|abc) bc in this order"</p> 
                  <p>I'd like to submit a patch for this issue. <img class="emoticon" src="/jira/images/icons/emoticons/smile.png" height="16" width="16" align="absmiddle" alt="" border="0"></p> 
                 </div>
