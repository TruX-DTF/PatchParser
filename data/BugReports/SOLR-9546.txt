                  <p>Here is an excerpt </p> 
                  <div class="code panel" style="border-width: 1px;">
                   <div class="codeContent panelContent"> 
                    <pre class="code-java">  <span class="code-keyword">public</span> <span class="code-object">Long</span> getLong(<span class="code-object">String</span> param, <span class="code-object">Long</span> def) {
    <span class="code-object">String</span> val = get(param);
    <span class="code-keyword">try</span> {
      <span class="code-keyword">return</span> val== <span class="code-keyword">null</span> ? def : <span class="code-object">Long</span>.parseLong(val);
    }
    <span class="code-keyword">catch</span>( Exception ex ) {
      <span class="code-keyword">throw</span> <span class="code-keyword">new</span> SolrException( SolrException.ErrorCode.BAD_REQUEST, ex.getMessage(), ex );
    }
  }
</pre> 
                   </div>
                  </div> 
                  <p><tt>Long.parseLong()</tt> returns a primitive type but since method expect to return a <tt>Long</tt>, it needs to be wrapped. There are many more method like that. We might be creating a lot of unnecessary objects here.</p> 
                  <p>I am not sure if JVM catches upto it and somehow optimizes it if these methods are called enough times (or may be compiler does some modifications at compile time)</p> 
                  <p>Let me know if I am thinking of some premature optimization</p>
