                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>In matrix multiplication, Sparse Matrices can easily turn dense and bloat memory, one fully dense column and one fully dense row can cause a sparse %*% sparse operation have a dense result. </p> 
                  <p>There are two issues here one with a quick Fix and one a bit more involved:</p> 
                  <ol> 
                   <li>in <tt>ABt.Scala</tt> use check the `MatrixFlavor` of the combiner and use the flavor of the Block as the resulting Sparse or Dense matrix type: 
                    <div class="code panel" style="border-width: 1px;">
                     <div class="codeContent panelContent"> 
                      <pre class="code-java">val comb = <span class="code-keyword">if</span> (block.getFlavor == MatrixFlavor.SPARSELIKE) {
              <span class="code-keyword">new</span> SparseMatrix(prodNCol, block.nrow).t
            } <span class="code-keyword">else</span> {
              <span class="code-keyword">new</span> DenseMatrix(prodNCol, block.nrow).t
            }
</pre> 
                     </div>
                    </div> <p> a simlar check needs to be made in the <tt>blockify</tt> transformation.</p></li> 
                  </ol> 
                  <ol> 
                   <li>More importantly, and more involved is to do an actual analysis of the resulting matrix data in the in-core <tt>mmul</tt> class and use a matrix of the appropriate Structure as a result.</li> 
                  </ol> 
                 </div> 
                </div> 
               </div>
