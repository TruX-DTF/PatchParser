                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>The scenario is a parent and child table with a cascade delete and triggers on both tables. Here are the steps to reproduce.</p> 
                  <p>First, compile TestFunctions.java and put it on the classpath:</p> 
                  <p>public class TestFunctions<br> {<br> public static void test(String str)<br> {<br> }<br> }</p> 
                  <p>Next, enter commands into interactive SQL:</p> 
                  <p>create table testtable (id integer, name varchar(20), primary key(id));</p> 
                  <p>create table testchild (<br> id integer<br> constraint fk_id references testtable on delete cascade,<br> ordernum int,<br> primary key(id));</p> 
                  <p>create procedure testproc (str varchar(20))<br> PARAMETER STYLE JAVA LANGUAGE JAVA EXTERNAL NAME 'TestFunctions.test';</p> 
                  <p>create trigger testtabletrigger after delete on testtable referencing old as old<br> for each row mode db2sql call testproc(char(old.id));</p> 
                  <p>create trigger testchildtrigger after delete on testchild referencing old as old<br> for each row mode db2sql call testproc(char(old.ordernum));</p> 
                  <p>insert into testtable values (1, 'test1');</p> 
                  <p>insert into testchild values (1, 10);</p> 
                  <p>delete from testtable where id = 1;</p> 
                  <p>The expected result is that deleting a row from "testtable" will cascade the delete to "testchild", and the triggers will be called for each delete. The actual result is that the delete is rolled back with the following error:</p> 
                  <p>Error: An attempt was made to put a data value of type 'java.lang.String' into a data value of<br> type 'INTEGER'.<br> SQLState: XCL12<br> ErrorCode: 30000</p> 
                  <p>There are no additional entries in the derby.log after the error. If only one trigger is used, or if the cascade is removed, then the delete will succeed.</p> 
                  <p>This issue was found while using SymmetricDS, which uses triggers to replicate tables between Derby databases.</p> 
                 </div> 
                </div> 
               </div>
