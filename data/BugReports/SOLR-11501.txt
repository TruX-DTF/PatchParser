                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>Solr should not parse local-params (and thus be able to switch the query parser) in certain circumstances. <em>Perhaps</em> it is when the QParser.getParser is passed "lucene" for the <tt>defaultParser</tt>? This particular approach is just a straw-man; I suspect certain valid embedded queries could no longer work if this is done incorrectly. Whatever the solution, I don't think we should assume 'q' is special, as it's valid and useful to build up queries containing user input in other ways, e.g. <tt>q= +field:value +{!dismax v=$qq}&amp;qq=user input</tt> and we want to protect the user input there similarly from unwelcome query parsing switching.</p> 
                 </div>
