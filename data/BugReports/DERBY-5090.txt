               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>This is my first issue report, so please be understanding if I'm posting the wrong thing, in the wrong place or in the wrong way. I just want to help. <img class="emoticon" src="/jira/images/icons/emoticons/smile.png" height="16" width="16" align="absmiddle" alt="" border="0"></p> 
                  <p>While iterating through a ResultSet, when accessing a BLOB field to read its contents via an InputStream, I noticed that:</p> 
                  <ul class="alternate" type="square"> 
                   <li>if the current ResultSet's has been "warmed up" by retrieving another column first, everything it's fine;</li> 
                   <li>if, on the other hand, you first-thing access the BLOB (and read other columns later), then upon reading the first byte out the InputStream bound to the BLOB field (ResultSet.getBinaryStream("col_name")) an IOException is thrown (and IOException's getMessage() method returns null).</li> 
                  </ul> 
                  <p>Following is an example, taken from a real application. The two code segments only differ in the fact that a SMALLINT &amp; VARCHAR read is done before/after the BLOB read.</p> 
                  <p>-<del>Working snippet</del>-<br> <span class="error">[...]</span><br> icRelPath<span class="error">[i]</span> = "imm" + File.separator + "ic" + "<em>" + rs.getShort("setIcone") + "</em>" + i + "." + rs.getString("estensione");<br> AutoCloseInputStream acis = new AutoCloseInputStream(rs.getBinaryStream("ic" + i));<br> if (rs.wasNull())<br> icRelPath<span class="error">[i]</span> = null;<br> else<br> {<br> //icRelPath<span class="error">[i]</span> = "imm" + File.separator + "ic" + "<em>" + rs.getShort("setIcone") + "</em>" + i + "." + rs.getString("estensione");<br> BufferedInputStream bis = new BufferedInputStream(acis);<br> int b = bis.read();//READS FINE<br> <span class="error">[...]</span></p> 
                  <p>-<del>Broken snippet</del>-<br> <span class="error">[...]</span><br> //icRelPath<span class="error">[i]</span> = "imm" + File.separator + "ic" + "<em>" + rs.getShort("setIcone") + "</em>" + i + "." + rs.getString("estensione");<br> AutoCloseInputStream acis = new AutoCloseInputStream(rs.getBinaryStream("ic" + i));<br> if (rs.wasNull())<br> icRelPath<span class="error">[i]</span> = null;<br> else<br> {<br> icRelPath<span class="error">[i]</span> = "imm" + File.separator + "ic" + "<em>" + rs.getShort("setIcone") + "</em>" + i + "." + rs.getString("estensione");<br> BufferedInputStream bis = new BufferedInputStream(acis);<br> int b = bis.read();//THROWS IOException WITH A null ERROR MESSAGE STRING<br> <span class="error">[...]</span></p> 
                 </div>
