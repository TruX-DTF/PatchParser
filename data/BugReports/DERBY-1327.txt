                 <div class="user-content-block"> 
                  <p>Using the following JRE with JIT on an identity column may be created with a wrong and very large START WITH value. When the problem occurs it affects not only the table being created, but also other tables that were created in previous transactions.</p> 
                  <p>For example attempting to create 1000 tables with identity columns the 126th table creation changes the start with value in sys.syscolumns to 41628850257395713 for ALL 125 tables. Attempts to insert into any of the tables cause<br> "SQL Exception: A truncation error was encountered trying to <br> shrink ... to length 12." </p> 
                  <p>This program will create up to 1000 tables until the problem <br> occurs<br> Note:</p> 
                  <ul class="alternate" type="square"> 
                   <li>The problem does not occur with -Xnojit (JIT OFF)</li> 
                   <li>The problem, when it occurs, changes not only the table being<br> created but all previous tables created. See output below. <br> Every thing was fine up until mytable126 and then all the <br> tables got changed to start with 41628850257395713 </li> 
                   <li>Problem occurs with autocommit on/off.</li> 
                   <li>The problem occurs after the create table but before the<br> commit.</li> 
                   <li>If the non-identity columns are removed the problem does not<br> reproduce.</li> 
                  </ul>

                  </ul>
