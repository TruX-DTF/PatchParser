                <h2 class="toggle-title">Description</h2>
               </div>
               <div class="mod-content">
                <div id="description-val" class="field-ignore-highlight"> 
                 <div class="user-content-block"> 
                  <p>I had to change a the primary key of a table and used ddlutils to do so. Ddlutils recreated the table to perform this task.<br> After the schema conversion the row data of the changed table were corrupted. <br> The values of the last table column were filled with values from other rows!</p> 
                  <p>After performing a few tests I could break down the problem to the SQL statement "INSERT INTO table SELECT FROM (...)"</p> 
                  <p>To reprocude the effect do the following:</p> 
                  <p>1. unpack attached database 'rmdb.zip'</p> 
                  <p>2. connect to the database with embedded driver <br> User: IGEL<br> Password: test </p> 
                  <p>3. read data of an example row from database<br> select * from stringrangenew where classname = 'x.xserver%.colordepth';</p> 
                  <p>result is <br> x.xserver%.colordepth 2 <span class="error">[16]</span> <span class="error">[24]</span> <span class="error">[32]</span></p> 
                  <p>4. now copy the data to the second table (column INSTANCENR has been added to this table)<br> INSERT INTO STRINGRANGENEW_ (CLASSNAME,FIRMWAREID,RVALUE) SELECT CLASSNAME,FIRMWAREID,RVALUE FROM STRINGRANGENEW;</p> 
                  <p>5. select data of example row from second table<br> select * from stringrangenew_ where classname = 'x.xserver%.colordepth';</p> 
                  <p>result is <br> x.xserver%.colordepth 2 -1 <span class="error">[CCW]</span><span class="error">[CW]</span><span class="error">[XX]</span></p> 
                  <p>-&gt; value of last column is not the same as in orignal table!</p> 
                  <p>Here some additional information i worked out during my tests:</p> 
                  <ul> 
                   <li>if you change the copy statement to include the additional column INSTANCENR, the copied data are correct.<br> delete from STRINGRANGENEW_;<br> INSERT INTO STRINGRANGENEW_ (CLASSNAME,FIRMWAREID, INSTANCENR, RVALUE) SELECT CLASSNAME,FIRMWAREID, -1, RVALUE FROM STRINGRANGENEW;</li> 
                  </ul> 
                  <ul> 
                   <li>if you select the rows only 'SELECT CLASSNAME,FIRMWAREID,RVALUE FROM STRINGRANGENEW', the result shows correct data</li> 
                  </ul> 
                  <p>Note: <br> The effect is not restricted to this row but also applies to other rows. But it's always the same rows, that get corrupted.</p> 
                 </div> 
                </div> 
               </div>
